## Priority 0: 修复文件夹转换闪退（必须立即修复！）

### 问题 1.1: 闪退原因分析

**可能的原因（按概率排序）**：

#### 原因 A: 内存溢出（最可能，80%）

**问题分析**：
```
场景：102 张图片，每张约 8MB
总大小：约 800+ MB

可能的问题：
1. 一次性加载所有图片到内存
2. 没有释放已处理完成的图片
3. 内存占用持续增长
4. 达到上限后崩溃
```

**验证方法**：
```
测试 1：只添加 5 张图片
- 如果不闪退 → 确认是内存问题

测试 2：添加 50 张图片
- 如果闪退 → 确认是数量相关

测试 3：添加 1 张大图片（20MB+）
- 如果闪退 → 确认是单个文件大小问题
```

**解决方案**：

**步骤 1: 实现逐个处理机制**
```
当前可能的逻辑（错误）：
for each file in files:
  加载图片到内存
  调用 API 转换
  等待结果

问题：
所有图片同时在内存中

正确逻辑：
for each file in files:
  加载图片到内存
  调用 API 转换
  等待结果
  释放内存  ← 关键！
  处理下一个
```

**步骤 2: 限制并发数量**
```
当前可能：
同时处理所有 102 张图片（并发）

改为：
每次只处理 3-5 张图片
处理完一批再处理下一批
```

**步骤 3: 及时释放资源**
```
关键点：
1. 图片 bytes 使用完立即清空
2. API 响应数据及时处理和释放
3. 不要在内存中保留已完成的图片数据
```

---

#### 原因 B: 异常处理不当（可能，15%）

**问题分析**：
```
场景：处理第一张图片时遇到错误
例如：
- 图片损坏
- API 返回异常
- 网络超时

当前处理：
可能没有 try-catch
导致未捕获异常使应用崩溃
```

**解决方案**：

**步骤 1: 添加全局异常捕获**
```
批量转换函数必须包裹在 try-catch 中

伪代码：
async function batchConvert() {
  try {
    for each file:
      try {
        处理单个文件
      } catch (单文件错误) {
        记录错误
        继续处理下一个
      }
  } catch (全局错误) {
    显示错误提示
    不要让应用崩溃
  }
}
```

**步骤 2: 错误隔离**
```
关键原则：
一个文件失败，不应影响其他文件

实现：
每个文件独立的 try-catch
失败的文件标记为失败
继续处理剩余文件
```

---

#### 原因 C: 后端处理大量文件崩溃（可能，3%）

**问题分析**：
```
场景：
一次性向后端发送 102 个文件
后端处理不过来
或超时
导致前端等待时崩溃
```

**验证方法**：
```
检查后端日志：
- 是否有错误
- 是否崩溃
- 是否超时
```

**解决方案**：

**步骤 1: 改为逐个发送**
```
当前可能：
一次性发送所有文件到 /api/batch-convert

改为：
循环调用 /api/convert
每次只发送一个文件
```

**步骤 2: 添加超时和重试**
```
单个请求超时：30 秒
失败重试：最多 2 次
超过重试次数：标记为失败，继续下一个
```

---

#### 原因 D: UI 线程阻塞（可能，2%）

**问题分析**：
```
场景：
在 UI 线程中进行大量计算
导致 UI 无响应
系统强制终止应用
```

**解决方案**：

**步骤 1: 使用异步处理**
```
确保所有耗时操作都是 async
不阻塞 UI 线程
```

**步骤 2: 分批更新 UI**
```
不要每处理一个文件就更新 UI
批量更新（例如每 5 个文件更新一次）
```

#### 阶段 3: 修复实施（1-2 小时）

**修复方案 A: 改为逐个处理（推荐）**

**实施步骤**：

**步骤 1: 修改转换逻辑**
```
原理：
一个一个处理文件
而不是同时处理所有文件

伪代码结构：
async function startConversion() {
  for (int i = 0; i < files.length; i++) {
    try {
      // 更新状态：处理中
      updateFileStatus(i, 'processing');
      
      // 读取文件
      final bytes = await readFile(files[i]);
      
      // 调用 API（单个）
      final result = await apiService.convertImage(
        file: files[i],
        format: format,
        quality: quality,
      );
      
      // 保存结果
      await saveResult(result);
      
      // 释放内存
      bytes = null;
      
      // 更新状态：完成
      updateFileStatus(i, 'completed');
      
    } catch (error) {
      // 标记失败，继续下一个
      updateFileStatus(i, 'failed', error: error);
    }
    
    // 小延迟，避免过快请求
    await Future.delayed(Duration(milliseconds: 100));
  }
}
```

**步骤 2: 限制并发数量（可选，更高级）**
```
如果想提高速度，可以 3-5 个并发

伪代码：
final pool = Pool(3);  // 最多 3 个并发

for (file in files) {
  await pool.withResource(() async {
    await processFile(file);
  });
}
```

**步骤 3: 添加进度更新节流**
```
不要每个文件都更新 UI
每 5 个文件更新一次

int processedCount = 0;
for (file in files) {
  await process(file);
  processedCount++;
  
  if (processedCount % 5 == 0) {
    updateProgress();  // 批量更新 UI
  }
}
```

## Priority 1: 修复垂直居中问题（视觉优化）

### 问题 2.1: 垂直居中未生效分析

**从截图 3 观察**：
```
拖拽区域高度：约 550px（估算）
内容（图标+文字）：约 200px
应该上下各留 175px

但实际：
上方留白：约 100px（偏少）
下方留白：约 250px（偏多）

结论：内容偏上了
```

**可能的原因**：

#### 原因 A: 使用了固定 padding（最可能）
```
可能的代码：
Container(
  padding: EdgeInsets.only(top: 100),  ← 固定了顶部
  child: Column(...)
)

问题：
固定 padding 会导致不居中
```

#### 原因 B: Column 没有设置居中对齐
```
可能的代码：
Column(
  children: [...],
  // 缺少这一行！
  // mainAxisAlignment: MainAxisAlignment.center,
)

问题：
Column 默认从顶部开始排列
```

#### 原因 C: 外层容器高度不对
```
可能的代码：
Container(
  height: 400,  ← 高度太小
  child: Center(child: Column(...))
)

问题：
容器高度不够，内容无法真正居中
```

---

### 问题 2.2: 修复步骤

**步骤 1: 检查当前布局代码**
```
需要找到拖拽区域的代码

关键词：
- DropTarget
- DottedBorder
- "拖拽图片或文件到这里开始转换"
```

**步骤 2: 确认布局结构**
```
理想的结构应该是：

Container (拖拽区域容器)
  height: 500px (或使用 Expanded)
  ↓
  Center (居中包裹)
    ↓
    Column (内容列)
      mainAxisAlignment: MainAxisAlignment.center
      crossAxisAlignment: CrossAxisAlignment.center
      ↓
      children:
        - Icon (云图标)
        - SizedBox(height: 24)
        - Text (主标题)
        - SizedBox(height: 12)
        - Text (副标题)
        - SizedBox(height: 16)
        - Text (特性说明)
```

**步骤 3: 实施修复**

**方案 A: 使用 Center + Column（推荐）**
```
结构：
Container(
  // 移除所有 padding
  child: Center(  ← 添加 Center
    child: Column(
      mainAxisSize: MainAxisSize.min,  ← 重要！
      children: [
        图标,
        间距,
        文字,
        ...
      ],
    ),
  ),
)
```

**方案 B: Column 居中对齐**
```
结构：
Container(
  child: Column(
    mainAxisAlignment: MainAxisAlignment.center,  ← 添加居中
    crossAxisAlignment: CrossAxisAlignment.center,
    children: [...]
  ),
)
```

**方案 C: 使用 Align**
```
结构：
Container(
  child: Align(
    alignment: Alignment.center,
    child: Column(
      mainAxisSize: MainAxisSize.min,
      children: [...]
    ),
  ),
)
```

**步骤 4: 验证居中效果**
```
检查方法：
1. 视觉检查：上下留白应该相等
2. 使用 Flutter DevTools 的 Layout Inspector
3. 调整窗口大小，内容应保持居中
```
