## 📋 文档说明

本文档最新反馈，针对空状态首页的 UI 问题和文件夹批量处理功能需求制定改进计划。

## 🔍 当前问题分析

### 截图 1 分析：空状态首页 UI 问题

**当前布局问题**：

```
┌──────────────────────────────────────────────────────┐
│                   顶部导航栏                          │
├──────────────────────────────────────────────────────┤
│                                                      │
│              [大量留白区域]                           │
│                                                      │
│         [文件夹图标]                                  │

│         点击选择文件或拖拽图片到下方区域                │
│                                                      │
│         [拖拽区域 - 虚线边框]                          │
│           [云上传图标]                                │
│           拖拽图片到这里                               │
│           支持 JPG·PNG·WebP·HEIC                     │
│                                                      │
│         [选择图片按钮]                                │

│                                                      │
├──────────────────────────────────────────────────────┤
│ [输出格式] [质量滑块]  [选择文件] [开始转换]           │
└──────────────────────────────────────────────────────┘
```

**存在的问题**：

1. **垂直空间利用率低**
   - 拖拽区域上下有大量空白
   - 内容集中在中间，上下留白过多
   - 整体布局不够紧凑

2. **拖拽区域尺寸不合理**
   - 高度不够，不够醒目
   - 内部元素间距过小
   - 没有充分利用可用空间

3. **视觉层次混乱**
   - "还没有添加图片"的提示和拖拽区域重复
   - 拖拽区域内外都有提示文字
   - 信息重复导致视觉混乱

4. **右侧区域完全空置**
   - 横向空间完全未利用
   - 可以考虑横向布局或增加功能区

5. **底部控制栏问题**
   - 空状态时底部控制栏没有必要显示
   - 或者应该禁用相关控件

## Priority 2: 文件夹批量处理功能（强烈建议）

### 改进 2.1: 添加"选择文件夹"功能

**当前行为**：
- 只有"选择图片"按钮
- 打开文件选择器，只能选择文件

**用户需求**：
- 需要"选择文件夹"功能
- 选择一个文件夹后，自动加载该文件夹下的所有图片
- 不递归子文件夹（只处理当前层级）

**实现目标**：

#### 步骤 1: 添加"选择文件夹"按钮

**位置选择**：

**方案 A：双按钮并列**
```
[选择图片] [选择文件夹]
```

**方案 B：下拉菜单**
```
[选择文件 ▼]
  - 选择图片
  - 选择文件夹
```

**方案 C：Tab 切换**
```
[ 图片文件 | 文件夹 ]
```

**推荐**：方案 A（双按钮并列）
- 最直观
- 功能明确
- 操作简单

---

#### 步骤 2: 实现文件夹选择逻辑

**调用系统 API**：

**Windows**：
- 使用 `FilePicker.platform.getDirectoryPath()`
- 返回文件夹路径

**流程**：
```
1. 用户点击"选择文件夹"
   ↓
2. 打开文件夹选择对话框
   ↓
3. 用户选择文件夹
   ↓
4. 获取文件夹路径
   ↓
5. 扫描该文件夹下的所有文件
   ↓
6. 筛选出图片文件（.jpg .png .webp .heic）
   ↓
7. 添加到文件列表
   ↓
8. 更新 UI
```

---

#### 步骤 3: 文件筛选规则

**支持的扩展名**：
```
.jpg, .jpeg, .png, .webp, .heic, .heif
```

**筛选逻辑**：
```
逻辑伪代码：

选中的文件夹路径 = folder_path

for each file in folder_path:
  if file.is_directory():
    跳过（不递归子文件夹）
  
  if file.extension in ['.jpg', '.jpeg', '.png', '.webp', '.heic', '.heif']:
    添加到文件列表
```

**大小写处理**：
- 扩展名不区分大小写
- .JPG, .Jpg, .jpg 都应识别

---

#### 步骤 4: 用户反馈

**处理中提示**：
```
点击"选择文件夹"后
↓
显示加载提示："正在扫描文件夹..."
↓
扫描完成
↓
提示："已添加 X 张图片"
```

**边界情况处理**：

| 情况 | 提示 |
|------|------|
| 文件夹为空 | "该文件夹中没有图片" |
| 只有非图片文件 | "未找到支持的图片格式" |
| 图片数量过多（>100） | "检测到 X 张图片，是否全部添加？" |
| 某些文件无法读取 | "已添加 X 张，跳过 Y 张无法读取的文件" |

---

#### 步骤 5: 性能优化

**大量文件处理**：

1. **异步扫描**
   - 使用异步方法避免 UI 卡顿
   - 显示加载指示器

2. **批量添加**
   - 一次性添加所有文件到列表
   - 避免多次 UI 更新

3. **限制数量**（可选）
   - 单次最多添加 100 张图片
   - 超过则提示用户分批处理
